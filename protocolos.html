PROTOCOLOS DE RED

Protocolos utilzados en redes informáticas y su importancia

Un protocolo de red es un conjunto de reglas y estándares que determinan cómo se envían, reciben y procesan los datos en una red. Asegura que los dispositivos de la red se comuniquen de manera efectiva y comprendan el formato y el contenido de la información intercambiada.

El protocolo TCP/IP es un conjunto de protocolos que gobierna la comunicación en Internet. TCP (Transmission Control Protocol) asegura la entrega confiable de datos, mientras que IP (Internet Protocol) se encarga de direccionar y enrutar los datos entre dispositivos. Juntos, garantizan la transmisión eficiente y correcta de información en redes.
Protocolo de Red y Protocolo TCP/IP


HTTP (Hypertext Transfer Protocol) es el protocolo que permite la transferencia de datos en la web, como páginas y archivos. HTTPS (HTTP Secure) es la versión segura de HTTP, que cifra los datos intercambiados entre el navegador y el servidor para proteger la privacidad y la integridad de la información.

FTP (File Transfer Protocol) es un protocolo utilizado para transferir archivos entre un cliente y un servidor a través de una red. Permite la carga y descarga de archivos de manera eficiente y es comúnmente usado para gestionar archivos en servidores web.
Protocolos HTTP/HTTPS y FTP

El Modelo de Referencia OSI (Open Systems Interconnection) es un marco de siete capas que estandariza las funciones de red para facilitar la interoperabilidad entre sistemas diferentes. Las capas son: Física, Enlace de Datos, Red, Transporte, Sesión, Presentación y Aplicación. Cada capa tiene funciones específicas que permiten la comunicación y transferencia de datos en una red.

El Modelo TCP/IP tiene cuatro capas: Acceso (maneja el hardware), Internet (rutea datos), Transporte (asegura la entrega) y Aplicación (gestiona la comunicación de aplicaciones). Establece los estándares para la comunicación en redes como Internet.
